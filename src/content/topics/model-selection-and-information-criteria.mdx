abstract: "Track 6 opened with linear regression as orthogonal projection (Topic 21) and continued with generalized linear models as IRLS on the exponential family (Topic 22). Topic 23 added penalization to stabilize both under collinearity and separation, with cross-validation as the default selector for the penalty parameter $\\lambda$. Topic 24 closes the track by formalizing the selection problem itself: given a candidate family $\\{\\mathcal{M}_1, \\mathcal{M}_2, \\dots\\}$, which model should we use, and by what criterion? The three main answers — AIC, BIC, and cross-validation — look superficially similar ($-2\\hat\\ell + \\text{penalty}$) but rest on distinct asymptotic guarantees. The featured §24.3 gives Akaike's theorem in full: AIC estimates the expected Kullback–Leibler divergence between the fitted model and the truth, with the $+2k$ penalty derived as the bias-correction term in the plug-in estimate of expected log-likelihood. §24.4 develops BIC as the Laplace approximation to the Bayesian marginal likelihood, with the $k\\log n$ penalty emerging from the second-order expansion — opening the on-ramp to Track 7's Bayesian machinery. §24.5 proves Stone's 1977 equivalence: LOO-CV and AIC select the same model asymptotically under Gaussian-homoscedastic errors, closing the loop on Topic 23's $\\lambda$-selection apparatus. §24.6 states Yang's 2005 incompatibility: no procedure can be both selection-consistent (BIC, when the true model is in the candidate set) and minimax-rate optimal for prediction (AIC, CV, in the misspecified-nonparametric regime). §24.7 connects LRT-based nested-model testing (Topics 18, 21, 22) to AIC via the identity $\\Delta\\mathrm{AIC} = 2q - 2\\Delta\\hat\\ell$. §24.8 extends the framework to penalized estimators via effective degrees of freedom ($\\operatorname{tr}(\\mathbf{H}_\\lambda)$ for ridge; $\\mathbb{E}[|\\mathcal{A}|]$ for lasso), discharging Topic 23 §23.8 Rem 20. §24.9 runs three workflows — polynomial-degree on simulated data, nested Poisson GLM, lasso-path AIC on prostate-cancer. §24.10 forward-maps to Track 7 (BIC → marginal likelihood → BMA → MCMC → DIC/WAIC/PSIS-LOO), Track 8 (nonparametric selection, Rademacher complexity), and formalml (post-selection inference, debiased lasso, cross-fitting)."

<DefinitionBlock number={1} title="Candidate model family">
A **candidate model family** is a finite collection $\{\mathcal{M}_1, \mathcal{M}_2, \ldots, \mathcal{M}_M\}$ of statistical models, each defining a parameter space $\Theta_k$ and a likelihood $L_k(\theta_k; \mathbf{y})$. The **model-selection problem** is to choose one model from this family, given the data $\mathbf{y}$, by a procedure that has a defensible asymptotic justification.
</DefinitionBlock>
